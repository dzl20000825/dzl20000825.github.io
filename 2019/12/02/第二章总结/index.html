第二章总结

#第二节具体讲解
1、面向对象：
符合人类思考习惯的一种编程思想。这种思想是执行者变为指挥者。提高了代码的开发效率以及可扩展性
2、面向过程：
一步一步从上到下实现功能。
3、程序的执行过程：
从main方法开始，从main方法的第一行，执行到最后一行
4、类的成员（组成部分）：
类由，属性（成员变量），构造方法，方法，静态方法，代码块，静态代码块构成
4.1、构造方法的形式：
方法名与类名一样
有空的构造方法与带参的构造方法
4.2、构造方法的作用：
Java可以通过无参或带参的构造方法完成赋值的初始化工作。
4.3、 //代码块，创建对象时被执行
{
System.out.println(“代码块被运行”);
}
4.5、//静态代码块，在类加载的时候执行，只执行一次
//用于类初始化
static{
System.out.println(“静态代码块运行”);
}
4.6、static
被static修饰的变量称为静态变量，静态变量被所有对象共有。被static修饰的方法称为静态方法。
静态方法可以直接通过类名调用，也可以通过对象调用。
静态方法不能使用this和super关键字，静态方法里边不可以直接使用非静态变量。
5、方法重载的判断依据：
必须在同一个类里面
方法名相同
参数列表不同
与方法返回值和方法修饰符没有任何关系
6、this
this代表当前对象，即正在调用该方法的对象
